# SalonEase 香港小型美容院管理系統

**純 PHP + API-First + 零 Framework**  
專業、簡單、高效率，專為香港小型美容院、醫美中心、SPA（3-15 人團隊）設計。

部署目標：https://salonease.ysk.hk

---

## 核心特色

- **純原生 PHP**：無 Laravel、CodeIgniter、Symfony、Composer（除極度必要）
- **API-First**：所有商業邏輯集中在 `/api/`，前端大量使用 fetch 呼叫
- **完整業務支援**：
  - 零售 + 服務 + 套票（療程卡）
  - 員工三類佣金（服務 / 零售 / 開單）
  - 房間容量 + 預約時間衝突避免
  - 熱感紙收據（58/80mm）+ A4 合約收據（純 CSS @media print，無 PDF 庫）
- **極致操作效率**：全面 Hotkey 支援，減少滑鼠依賴
- **專業現代 UI**：Tailwind CSS（CDN）+ Alpine.js（CDN）+ 原生 JS
- **香港在地化**：所有介面、錯誤、報表皆用繁體中文（香港習慣用語）
- **共享主機友好**：簡單部署，無複雜依賴

## 快速開始（本地開發）

1. 複製 `config.example.php` 為 `config.php` 並填入資料庫資訊
2. 建立 MySQL 資料庫（建議 utf8mb4_unicode_ci）
3. 執行 `sql/schema.sql` 建立所有表格
4. （可選）執行 `sql/seeds.sql` 插入測試資料
5. 將專案目錄設為 web root 或使用 PHP 內建伺服器：
   ```bash
   php -S localhost:8000
   ```
6. 瀏覽 http://localhost:8000 即可登入（預設帳號見 seeds）

## 部署與注意事項

### 推薦部署方式（一鍵安裝）

SalonEase 已提供專業級一鍵安裝工具，**強烈建議使用此方法**，而非手動執行 SQL。

#### 步驟：
1. 將專案上傳到網站根目錄（或子目錄）
2. 建立 MySQL 資料庫，並在 `config.php` 填入正確連線資訊（**切勿將 config.php 加入 Git**）
3. 瀏覽 `https://你的域名/install.php`
4. 選擇安裝模式：
   - **乾淨版**：只建立管理員帳號 + 基本設定（適合正式上線）
   - **完整十年模擬數據版**：自動產生 2015–2025 年真實營運記錄（適合展示、培訓、測試）
5. 安裝完成後**立即刪除 `install.php`**（系統會提示）
6. 登入後立即修改管理員密碼
7. 前往「系統設定」完善店舖資訊、打印設定及佣金比率

#### 日後升級流程（強烈建議閱讀）

SalonEase 設計了專業的一鍵升級機制，未來任何資料庫結構更新（新增欄位、索引、新表、seed 更新等）都可以安全、一鍵完成。

**升級步驟：**
1. 登入系統（必須是 admin 或 manager 角色）
2. 瀏覽 `https://你的域名/upgrade.php`
3. 系統會自動掃描 `migrations/` 資料夾，列出所有待執行的更新
4. 按下「開始一鍵升級」按鈕
5. 系統會逐一執行 migration，並以交易保護（失敗會自動回滾）

**如何新增 migration（給開發者）：**
- 在 `migrations/` 資料夾建立新檔案，例如 `003_add_customer_points.php`
- 檔案格式範例：
  ```php
  return [
      'description' => '為 customers 表新增積分欄位',
      'up' => function($pdo) {
          // 安全、冪等的 SQL
          try {
              $pdo->exec("ALTER TABLE customers ADD COLUMN points INT DEFAULT 0");
          } catch (PDOException $e) {
              // 忽略已存在欄位的錯誤
          }
      }
  ];
  ```
- 放入後，其他環境只要去 `upgrade.php` 就能自動套用

**升級前後檢查清單：**
- 升級前：備份資料庫
- 升級後：檢查 `upgrade.php` 是否顯示「已是最新版本」
- 如有自訂 migration，建議先在測試環境驗證

**強烈建議**：
- 生產環境務必定期備份資料庫（尤其是 `sales`、`commissions`、`customer_packages` 表）
- 使用 HTTPS
- 共享主機建議開啟 PHP OPcache

### 常見問題 FAQ

#### 部署相關
**Q: 出現「Session ini settings cannot be changed when a session is active」？**  
A: 這是因為 `config.php` 的 session 設定在 session 啟動後才執行。已加入保護機制（session_status 檢查）。請確保使用最新版本的 `config.php`。

**Q: 「Timezone ID 'Asia/Hong Kong' is invalid」？**  
A: 正確寫法是 `Asia/Hong_Kong`（用底線）。已修正 `config.php` 及 `config.example.php`，並加入防禦性處理。

**Q: install.php 出現 Fatal Error 或安裝失敗？**  
A: 最常見原因是 `config.php` 資料庫連線錯誤。請檢查 DB_HOST、DB_NAME、DB_USER、DB_PASS 是否正確，以及 MySQL 使用者權限。

**Q: 「Cannot redeclare get_current_user()」？**  
A: 這是函式名稱與 PHP 內建函式衝突，已將函式重新命名為 `get_logged_in_user()`。請更新所有相關檔案。

#### 使用相關
**Q: 手機版顯示不正常或操作困難？**  
A: 系統已大幅優化手機/平板響應式（POS、表格、Modal、按鈕等）。建議使用最新版本，並清除瀏覽器快取。

**Q: upgrade.php 顯示沒有更新，但我想手動執行某些改動？**  
A: migration 機制是冪等的。如需手動調整，請在 `migrations/` 資料夾新增新的 migration 檔案，然後再跑一次 upgrade.php。

**Q: 安裝後想刪除示範數據？**  
A: 建議直接在資料庫手動刪除，或使用「完整十年模擬數據版」只作測試用途，正式上線請選擇「乾淨版」安裝。

**Q: 如何備份？**  
A: 強烈建議定期使用 hosting 控制台或 `mysqldump` 備份整個資料庫，特別是 `sales`、`commissions`、`customer_packages` 表。
