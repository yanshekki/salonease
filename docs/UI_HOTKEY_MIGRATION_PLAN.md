# SalonEase 新 UI + Hotkey 完整設計方案

**日期**：2026 年
**背景**：從 Tailwind CDN + 自訂 CSS 逐步遷移到 Bootstrap 5 + 自訂主題。目標是現代企業級、專業乾淨、有質感、適合美容行業、高效率操作。

---

## 1. 新 UI 整體設計風格建議

### 核心風格定位
- **現代企業級美容管理系統**：不是可愛風，而是專業、可靠、高效率。
- **乾淨專業**：大量留白、清晰層次、一致間距。
- **高效率操作**：大按鈕、清晰焦點、熱鍵優先、減少滑鼠依賴。
- **美容行業質感**：柔和中性色調 + 優雅強調色，避免過度粉嫩或醫療白。

### 推薦配色方案（Bootstrap 5 自訂主題）
```css
--bs-primary: #2C2C2E;        /* 深灰黑 - 主文字/按鈕 */
--bs-secondary: #8FA68F;      /* 鼠尾草綠 - 強調色 */
--bs-success: #8FA68F;
--bs-danger: #C62828;
--bs-warning: #C9A86C;        /* 暖金色 */
--bs-info: #6C7A89;

--bs-body-bg: #FDF8F3;        /* 暖米白 - 背景 */
--bs-body-color: #2C2C2E;
--bs-border-color: #EDE5DC;   /* 柔和米色邊框 */
--bs-card-bg: #FFFFFF;
--bs-table-hover-bg: #F8F5F0;
```

### 排版與元件原則
- **字體**：Noto Sans TC（已使用）+ Inter 作為輔助
- **圓角**：統一使用 `rounded-2` 或 `rounded-3`（Bootstrap 預設可微調）
- **陰影**：輕微專業陰影（`shadow-sm`），避免過重
- **按鈕**：主要用實心（Primary），次要用 outline。所有按鈕高度至少 38px（手機友好）
- **表格**：乾淨、 zebra 輕微、hover 清晰
- **Modal**：大而清晰，標題粗體，底部按鈕區分明顯
- **表單**：標籤在上方，輸入框有足夠內距，錯誤狀態清晰（紅色邊框 + 文字）
- **導航**：頂部固定導航 + 側邊或頂部次導航（視頁面）

### 適合美容行業的細節
- 使用柔和綠色作為強調（代表專業護理）
- 重要操作用金色/暖色點綴（收銀、打印）
- 低庫存用清晰紅色警示
- 套票扣減用紫色或深綠區隔

## 3. 可能遇到的技術難點及解決建議（純 PHP 環境）

1. **大量手動替換 Tailwind classes**
   - 難點：頁面多，容易遺漏。
   - 解決：建立轉換對照表 + 由簡單頁面開始（login → dashboard → 設定）。

2. **Bootstrap 與現有 Alpine.js 衝突**
   - 解決：優先使用 Bootstrap 內建 JS（Modal, Dropdown），只在複雜互動保留 Alpine。

3. **手機響應式已做的工作可能需要調整**
   - 解決：Bootstrap 有良好 grid 系統，可逐步替換 `lg:grid-cols-12` 類別。

4. **自訂顏色主題在 Bootstrap 5 的實作**
   - 使用 `:root` CSS Variables 覆蓋 `--bs-*` 變數，最乾淨。

5. **熱鍵在輸入框內的衝突**
   - 已有 `if (['INPUT','TEXTAREA'].includes(tag))` 保護，繼續沿用。

6. **Git 衝突風險高**
   - 強制每小 chunk commit + push。

## 4. 建議開發順序 / 實施計劃（Phase）

**重要原則**：每完成一個 Phase（或 Phase 內大 chunk）後，**必須** `cd /home/ki/文件/salonease && git add . && git commit -m "..." && git push origin main`

### Phase 1: Bootstrap 基礎 + 品牌主題（1-2 天）
- 替換 header.php 為 Bootstrap CDN + 自訂主題
- 建立 `assets/css/bootstrap-custom.css`
- 轉換全域元件（按鈕、輸入框、卡片、表格、Modal 基礎樣式）
- **交付物**：可運行的 Bootstrap 版本首頁 + 設定頁

### Phase 2: 核心操作頁面遷移 + Hotkey 強化（3-5 天）
- 優先遷移：POS、預約、客戶
- 同時實施新 Hotkey 設計方案（全局 + 模組）
- 加入畫面提示（? 幫助 Modal 加強）
- **交付物**：POS、預約、客戶在 Bootstrap 下可用 + 完整熱鍵支援

### Phase 3: 其餘頁面 + 文件更新（2-3 天）
- 遷移報表、佣金、員工、產品等
- 更新 README 熱鍵文件
- **交付物**：全站可用於 Bootstrap

### Phase 4: 測試、優化、移除 Tailwind（2 天）
- 全面測試手機/平板
- 移除 Tailwind CDN 及舊 CSS
- 最終清理
- **交付物**：乾淨的 Bootstrap 版本，無 Tailwind 依賴
